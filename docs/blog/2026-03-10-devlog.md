# 2026-03-10 开发日志（补记）

原始开发日期：2026-03-10  
补记时间：2026-03-11 10:55

![2026-03-10 开发日志代表图](/blog/devlog-2026-03-10.svg)

## 昨天实际做了什么

1. 梳理 Context Menu 组件在示例和文档中的行为描述，统一术语。
2. 修正部分“看起来一样、其实不一样”的参数说明，减少误用概率。
3. 核对示例页面入口，确保文档里的链接能直达可运行示例。

## 当时遇到的小坑

- 右键菜单在不同场景下的触发语义容易混淆。
- 文档里写“推荐”，业务代码里却写“强制”，这俩是两种宇宙。

## 补记复盘

这天最大的收获不是写了多少代码，而是把“理解成本”压低了。  
组件是否高级，很多时候取决于别人第一次打开文档时有没有想关掉页面。

## 接下来怎么接

1. 继续把组件文档与 API 页做一轮交叉校验。
2. 为常见误用场景补上 Troubleshooting 条目。
